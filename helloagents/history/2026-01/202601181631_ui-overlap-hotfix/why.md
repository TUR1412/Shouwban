# 变更提案: UI 重叠/塌陷热修复

## 需求背景
- 现状：前端页面存在“重叠 / 折叠 / 塌陷”等视觉问题，影响阅读与交互。
- 影响：首页首屏 Hero、策展区块，以及 Header 在移动端的交互叠加时会出现遮挡/布局异常。
- 目标：修复结构性 HTML 闭合错误与 Header 交互互斥逻辑，提升页面布局稳定性；并通过版本号 bump 确保 PWA/缓存可正确更新。

## 变更内容
1. 修复首页 `index.html` 的结构性多余闭合标签，保证 DOM 树稳定、布局计算一致。
2. Header 移动端“菜单 / 搜索 / 下拉”互斥，避免 overlay 叠加遮挡。
3. 统一缓存穿透版本号 bump，触发 SW 与静态资源更新。

## 影响范围
- **模块:** UI / Header / 主页
- **文件:** `index.html`、`scripts/main.js`、`sw.js`、`*.html`（版本号）、`Task_Status.md`、`helloagents/*`（文档）
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 首页布局稳定性
**模块:** UI
修复首页在不同浏览器/分辨率下出现的区块重叠/塌陷，确保 Hero、策展等区块正常排版。

#### 场景: 进入首页（首屏）
- 条件：打开 `index.html`
- 预期结果：Header、Hero、策展区块不重叠，不出现“折叠到一行/高度为 0”等异常。

#### 场景: 首页滚动与 sticky header
- 条件：滚动页面，触发 sticky header
- 预期结果：Header 行为稳定，不遮挡正文，区块间距保持一致。

### 需求: Header 移动端交互互斥
**模块:** UI
确保移动端 Header 的“菜单 / 搜索 / 下拉”互斥，避免多个浮层同时打开导致遮挡/重叠。

#### 场景: 先打开搜索，再打开菜单
- 条件：点击 Header 搜索按钮打开搜索栏；再点击菜单按钮
- 预期结果：菜单打开时自动关闭搜索栏并收起下拉菜单，不发生重叠。

#### 场景: 先打开菜单，再打开搜索
- 条件：点击菜单按钮打开导航；再点击搜索按钮
- 预期结果：搜索打开时关闭菜单与下拉菜单（保持既有逻辑）。

## 风险评估
- **风险:** 修改 Header 交互可能影响少数边缘交互序列（例如快速连点）。
- **缓解:** 保持改动为“互斥关闭”的最小增量；执行 `npm run verify` 与 `npm test` 做回归验证。
